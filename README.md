# Сайт интернет-магазина, блог
Сайт интернет-магазина, реализованный на Python с применением фреймворка Django. Собран из практических работ курса
**"Python-фреймворк Django"** от **SkillBox**. Дополнен функциональностями (отмечено комментариями в коде). Некоторые
особенности реализации:
- Основной язык локализации - английской. Но часть функционала локализована на два языка: русский и английский.
- Лишь часть функционала работы с товарами и заказами покрыта тестами.

Разработанный сайт разворачивается в виде [контейнера Docker](#контейнеризация-в-docker).

Основное предназначение - сайт магазина товаров. Но есть и дополнительные функциональности, не связанны с ним.
Перечень функционала:
- Регистрация новых пользователей, их авторизация.
- Собственная форма просмотра групп пользователей.
- Работа с товарами.
- Работа с заказами.
- Блог со статьями. RSS-лента статей.
- API для работы с товарами и заказами.
- API просмотра групп пользователей.
- Документация Swagger по спецификации OpenAPI.
- Карта сайта. Включает в себя приложения [ShopApp](#shopapp) и [BlogApp](#blogapp)
- Импорт и экспорт данных товаров и заказов в **Django Admin**.
- Логирование.

Управление зависимостями управляется при помощи `Poetrty`. Для нужд отладки сохранён и упрощённый вариант с применением
файла `requirements.txt`.

## Функционал приложений
На верхнем уровне проекта `mysite`, кроме приложений, хранятся служебные каталоги, используемые в этих приложениях.

|      **Каталог**      | <div align="center">**Применение**</div>         |
|:---------------------:|--------------------------------------------------|
| additional_resources  | Пример загружаемого файла заказов в формат CSV.  |
|   mysite/templates    | Базовый шаблон форм.                             |
|    mysite/database    | База данных SQLite.                              |
|     mysite/locale     | Файлы локализации приложений.                    |
| mysite/media_uploads  | Файлы, загруженные пользователями (изображения). |

### MyAuth
Регистрация и авторизация пользователей. Реализует работу с профилями пользователей `Profile`. 

### ShopApp
Основной функционал. Здесь реализована работа с товарами `Product` и заказами `Order`. А также API работы с ними.
Раздел частично локализован на русский язык. Обе модели подключены к **Django Admin**.

Для заполнения тестовыми данными предусмотрена команда `load_shopapp_test_data`. Также предусмотрены отдельные команды
для демонстрации возможностей приложения.

### BlogApp
Дополнительный функционал. Отдельный блог для учёта авторов, статей, категорий статей, тегов. Реализована RSS-лента
статей.

Для заполнения тестовыми данными предусмотрена команда `load_blogapp_test_data`.

### RequestDataApp
Тестирование отправки файлов по HTTP. Реализирует форму загрузки файла.

### MyApiApp
Тестирование работы с API. Реализирует, в том числе, API получения групп пользователей.

## Контейнеризация в Docker
Предусмотрена сборка сайта в контейнер Docker. В файле **Docker Compose** представлены несколько наборов настроек.
Необязательные настройки по умолчанию закомментированы. Целевое имя сервиса - **App**.
- По умолчанию используется только сервис. Логирование в файлах JSON.
- В закомментированных настройка конфигурация для использования логирования с **Grafana Loki**.
- Для нужд отладки в комментариях есть настройки запуска без **Gunicron**.

Предполагается, что создание новой базы и создание супер-пользователя будет выполняться из контейнера Docker.
Информацию об этом и нужные команды также можно найти в файле Docker Compose.
